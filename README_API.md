API-et skal håndtere spillrundene. Klienten sender inn symbolet som velges og blir validert før den behandles videre.
GET/games returner data for spillrunder (foreløpig bare satt opp en test hvor resultatet er hardkodet). GET/games/:gameId gir respons for detaljer om en spesifikk spillrunde (gir/viser bare en placeholder for nå). POST/games lager et spill hvis spilleren sender inn et gyldig symbol (gir/viser også bare en placeholder for nå).
